# LINE Bot Go 版本 - 一鍵啟動指南

## 🚀 **一鍵啟動方式**

### **方式 1: 雙擊啟動 (最簡單)**
```
📁 雙擊文件: run_go.bat
```
只需要雙擊 `run_go.bat` 文件即可！

### **方式 2: 命令行啟動**
```bash
# 在 line_bot_go 目錄下執行
run_go.bat
```

### **方式 3: 直接運行**
```bash
# 直接運行可執行文件
line-bot-go.exe
```

## 🔧 **智能啟動腳本功能**

`run_go.bat` 腳本提供以下功能：

### **自動檢測**
- ✅ 檢查可執行文件是否存在
- ✅ 檢查端口 5003 是否被占用
- ✅ 測試現有服務是否正常運行

### **智能處理**
- 如果服務已運行：顯示訪問地址
- 如果端口被占用：提供診斷信息
- 如果文件不存在：提示先編譯

### **用戶友好**
- 🟢 中文界面說明
- 🔍 自動健康檢查
- 📊 提供訪問地址
- 🛑 清晰停止方式

## 📋 **完整啟動流程**

### **步驟 1: 檢查編譯**
```bash
# 檢查可執行文件
dir line-bot-go.exe
```

### **步驟 2: 一鍵啟動**
```bash
# 雙擊或執行
run_go.bat
```

### **步驟 3: 驗證運行**
```
🌐 訪問地址: http://localhost:5003/
📊 健康檢查: http://localhost:5003/health
🛑 按 Ctrl+C 停止服務
```

## 🎯 **相比 Python 版本的優勢**

| **依賴檢查** | 自動檢測 | 需手動檢查 |
| **端口管理** | 智能占用檢測 | 手動處理 |

## 🔍 **故障排除**

### **問題 1: 文件不存在**
```
❌ line-bot-go.exe 不存在
💡 請先執行 build_and_run.bat 編譯項目
```

**解決方案：**
```bash
build_and_run.bat
```

### **問題 2: 端口占用**
```
⚠️ 端口 5003 已被占用
💡 LINE Bot Go 可能已在運行
```

**解決方案：**
```bash
# 檢查端口占用
netstat -an | findstr :5003

# 停止現有服務
taskkill /f /im line-bot-go.exe
```

### **問題 3: 服務無法訪問**
```
❌ 服務無法訪問，請檢查問題
```

**解決方案：**
```bash
# 檢查服務狀態
curl -s http://localhost:5003/health

# 查看錯誤日誌
# 在啟動窗口查看輸出信息
```

## 🚀 **極簡使用指南**

### **首次使用**
1. 確保有 `line-bot-go.exe` 文件
2. 雙擊 `run_go.bat`
3. 看到 "正在啟動服務..." 即成功

### **日常使用**
1. 雙擊 `run_go.bat`
2. 看到訪問地址即成功
3. 按 Ctrl+C 停止

### **緊急情況**
如果出現問題，運行：
```bash
# 停止所有相關進程
taskkill /f /im line-bot-go.exe
taskkill /f /im nginx.exe

# 重新啟動
run_go.bat
```

## 🎊 **總結**

Go 版本的啟動方式確實比 Python 版本更簡單：
- **Python 版本**: 需要多個步驟，依賴檢查，複雜命令
- **Go 版本**: 雙擊即用，智能檢測，一鍵啟動

**只需要記住一個命令：`run_go.bat` 🎯**
